New Relic과 [New Relic One](/docs/new-relic-one/introduction-new-relic-one) 에 오신 것을 환영합니다!

우리는 디지털 소스에서 데이터를 가져오고 스택에서 무슨 일이 일어나고 있는지 이해하는 데 도움이 되는 관찰 가능성 플랫폼이므로 환경을 사전 예방적이고 예측적으로 개선할 수 있습니다. 클라우드에서 서비스를 실행하는 컨테이너에서 모바일 웹사이트의 장바구니 버튼에 이르기까지 애플리케이션 및 장치의 전체 그림을 제공합니다. New Relic을 사용하면 더 나은 소프트웨어를 구축하고 고객에게 가치를 제공할 수 있습니다.

[New Relic 계정에 가입하고](https://newrelic.com/signup) (영원히 무료입니다!) 모니터링 서비스를 설치하고 [데이터 작업을 시작하세요](/docs/data-apis/get-started/nrdb-horsepower-under-hood) . 깨끗하고 빠릅니다!

## 브라우저 모니터링 설치 [#browser-install]

New Relic의 브라우저 모니터링은 페이지별 인기도 및 사용자 만족도(Apdex), JavaScript 오류 분석, 상호 작용 이벤트, AJAX 요청 등과 같은 전체 페이지 수명 주기 및 세션 데이터를 측정합니다.

[브라우저 모니터링 설치를](/docs/browser/new-relic-browser/installation/install-new-relic-browser-agent) 참조하십시오.

## 합성 모니터 설치 [#synthetics]

[종합 모니터링](/docs/synthetics) 을 통해 웹사이트 또는 API 엔드포인트를 사전에 모니터링하여 콘텐츠가 완전히 작동하는지 확인할 수 있습니다. 스크립팅된 브라우저 테스트를 전 세계 위치에서 사이트로 보내어 콘텐츠가 어디서나 항상 작동하는지 확인합니다.

Synthetics는[전용 미니언](/docs/synthetics/new-relic-synthetics/private-locations/install-configure-private-minions) 기능을 제외하고는 설치가 필요하지 않습니다.

## 문제점 해결 [#troubleshooting]

에이전트를 설치하고 약간의 트래픽을 생성하고 몇 분 정도 기다린 후에 New Relic UI에서 데이터를 보기 시작해야 합니다. 데이터가 표시되지 않으면 [문제 해결 절차](/docs/agents/manage-apm-agents/troubleshooting/not-seeing-data) 를 따르십시오.

도움이 더 필요하세요? [연락처 지원!](/docs/new-relic-solutions/solve-common-issues/find-help-use-support-portal)

## 새로운 유물 하나 사용 [#use-nr]

모든 설정? 그런 다음 New Relic One에서 데이터 탐색을 시작하십시오!

* [기본 UI 기능](/docs/new-relic-one/use-new-relic-one/ui-data/basic-ui-features/) 을 사용해 보세요.
* [New Relic Explorer](/docs/new-relic-one/use-new-relic-one/ui-data/new-relic-one-entity-explorer) 를 사용하여 모니터링된 개체를 검색합니다.
* [데이터 탐색기](/docs/query-your-data/explore-query-data/data-explorer/introduction-data-explorer) 를 사용하여 쿼리 없는 환경에서 데이터를 확인하세요.
* NRQL(우리의 쿼리 언어) 전문 지식을 사용하여 [쿼리 빌더](/docs/specify-data-chart#use-advanced) 에서 사용자 지정 차트를 작성하십시오.
* [대시보드](/docs/dashboards/new-relic-one-dashboards/get-started/introduction-new-relic-one-dashboards) 에 대해 알아보세요.
